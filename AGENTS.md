### СПЕЦИФИКАЦИЯ
Разработать веб приложение над существующим API
Это мобильная фомра для оформления заказа
Не переусложнять, сделать один экран

Смысл там в следующем - ввод токена, авторизация. Затем открывается доступ к остальным полям. Они должны быть подключены к API
Все это на одном экране, ничего не должно пропадать и тд. Если к чему-то сейчас нет доступа(например пользователь еще не авторизирован) - просто оставь с disabled state
